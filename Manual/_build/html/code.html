synopsis:This module defines the component class, which describe common properties for particle/field components

:author:: Andre Lessa <lessa.a.p@gmail.com>

class component.Component(label, Type, dof, mass, decays=<AuxDecays.DecayList object>, sigmav=0.0, source=0.0, coherentAmplitute=0.0)[source]¶

Bases: object

Main class to hold component properties.

Parameters:
  • label – label for the component (can be anything)
  • ID – a proper name for defining the component (must belong to IDs)
  • Type – must be weakthermal, thermal or CO (weakly coupled thermal, thermal or coherent oscillating field)
  • active – if component is active or not (True/False)
  • dof – +/- Number of degrees of freedom (must be negative for fermions and positive for bosons)
  • Tdecay – stores the temperature at which the particle starts to decay (approximate)
  • Tosc – stores the temperature at which the particle starts to oscillate (if Type = CO)
  • Tdecouple – stores the temperature at which the particle decouples (if Type = thermal/weakthermal)
  • evolveVars – Stores the values of the variables used for evolution
getBRX(T)[source]¶

Get the decays for a given temperature T

getNTh(T, nratio, comp=None)[source]¶

Computes the effective thermal number density at temperature T: If comp = None:

Nth = neq[self]*sum_{decays} BR(self->a + b +...)*(n[a]/neq[a])*(n[b]/neq[b])* ...
If comp = component:
Nth = (neq[self]/Norm)*sum_{decays} N_component BR(self-> component + b +...)*(n[a]/neq[a])*(n[b]/neq[b])*...
Parameters:
  • T – temperature (allows for T-dependent BRs)
  • nratio – Dictionary with ratios of number density to the equilibrium number density.
  • comp – Component (if needed) to compute the weight X->Y+...
getOscAmplitute(T)[source]¶

Computes the oscillation amplitude for the component at temperature T. If the component is of Type thermal, returns None.

Checks if the component has fully decayed

isOscillating(T, H)[source]¶

Checks if the component is coherent oscillating. If it is a thermal field, it returns False. If it is a CO oscillating field and if self.active AND 3*H(T) < self.mass, returns True.

nEQ(T)[source]¶

Returns the equilibrium number density at temperature T. Returns zero for non-thermal components

rEQ(T)[source]¶

Returns the ratio of equilibrium energy and number densities at temperature T, assuming chemical potential = 0.

setInitialCond(T)[source]¶

Set initial conditions for component at temperature T, assuming the energy density is dominated by radiation.

width(T)[source]¶

Returns the component width at temperature T.

boltzSolver File¶

synopsis:This module contains the main methods for solving the Boltzmann equations
author:Andre Lessa <lessa.a.p@gmail.com>
boltzSolver.Evolve(compList, T0, TF, omegaErr=0.01)[source]¶

Evolve the components in component list from the re-heat temperature T0 to TF For simplicity we set R0 = s0 = 1 (with respect to the notes). Returns a list with components, where the evolveVars hold the evolution of each component. The last element of the list is a simple array with the evolution of NS. omegaErr is the approximate relative error for Omega h^2.

boltzSolver.goodCompList(compList, T0)[source]¶

Checks if the list of components satisfies the minimum requirements

boltzSolver.mySolve(xf, boltz_eqs, rtol, atol, verbosity=50)[source]¶

Sets the main options for the ODE solver and solve the equations. Returns the array of x,y points for all components. If numerical instabilities are found, re-do the problematic part of the evolution with smaller steps

class boltzEqs.BoltzEqs(compList, x0=None, y0=None, sw=None)[source]¶

Bases: assimulo.problem.Explicit_Problem

Class to hold the Boltzmann equations

handle_event(solver, event_info)[source]¶

Activate/de-activate components when a discontinuous transition happens and sets the new initial conditions. Possible discontinuities are: a CO component starts to oscillate.

rhs(x, y, sw)[source]¶

Defines the derivatives of the y variables at point x = log(R/R0). sw = [True/False,...] is a list of switches to activate/deactivate components If a component is not active it does not evolve and its decay and energy density does not contribute to the other components. For simplicity we set R0 = s0 = 1 (with respect to the notes).

state_events(x, y, sw)[source]¶
Checks for a discontinuous transition happened. Possible discontinuities are:
  • a CO component starts to oscillate * a particle has decayed and its number density is effectively zero

The transition vector must be zero when such a transition occurs. Also updates the absolute tolerance of the solver, if the variable’s values go below it.

updateValues(x, y, sw)[source]¶

Replace the variables in self.components and set the new initial conditions.

AuxFuncs File¶

synopsis:This module provides auxiliary functions

AuxFuncs.Hfunc(T, rhov, sw)[source]¶

Compute the Hubble parameter, given the variables x=log(R/R0) and ni, rhoi and NS=log(S/S0)

AuxFuncs.gSTARSexact(T)[source]¶

Computes the number of relativistic degrees of freedom for computing the entropy density, including the full MSSM spectrum, except for the lightest neutralino.

AuxFuncs.gSTARexact(T, interpol=True)[source]¶

Computes exactly the number of relativistic degrees of freedom in thermal equilibrium with the thermal bath. interpol turns on/off the interpolation around the QCD phase trasition region.

AuxFuncs.getDNeff(comp, TF)[source]¶

Computes the contribution from component comp to the number of effective neutrinos at temperature TF. Can only be used after the Boltzmann equations have been solved and the solutions stored in comp.evolveVars. Gives zero if T > 1 MeV (where neutrinos are still coupled).

AuxFuncs.getDataFrom(dataFile)[source]¶

Reads a datafile generated by printData, printSummary and printParameters and returns a dictionary with the information.

AuxFuncs.getFunctions(pclFile)[source]¶

Computes the g*(T), g*s(T) and temperature functions and saves them to a pickle file. Ignores all BSM effects to these functions :param pclFile: Name of pickle file to dump the functions

AuxFuncs.getOmega(comp, rho, n, T)[source]¶

Compute relic density today, given the component, number density and energy density at temperature T.

AuxFuncs.getPressure(mass, rho, n)[source]¶

Computes the pressure for a component, given its mass, its energy density and its number density

AuxFuncs.getTemperature(x, NS)[source]¶
AuxFuncs.getTexact(x, NS)[source]¶

Computes the temperature for the thermal bath from xeff = NS - 3*x, where x = log(R) and NS = log(S).

AuxFuncs.gstarFunc(x, dof)[source]¶

Auxiliary function to compute the contribution from a single particle to gSTAR. x = mass/T, dof = number of degrees of freedom (positive/negative for bosons/fermions)
